# Examples — Apex Outbound Email Patterns

## Example 1: Send from an Org-Wide Address with a routed Reply-To

Most production sends need: a shared "from" address, replies routed
to a different inbox, and an audit log on the related record.

```apex
public class TransactionalSender {
    public static void sendOrderConfirmation(Order__c order) {
        OrgWideEmailAddress owe = [
            SELECT Id FROM OrgWideEmailAddress
            WHERE Address = 'orders@acme.com' LIMIT 1
        ];

        Messaging.SingleEmailMessage msg = new Messaging.SingleEmailMessage();
        msg.setOrgWideEmailAddressId(owe.Id);
        msg.setReplyTo(new String[]{ 'support@acme.com' });
        msg.setToAddresses(new String[]{ order.CustomerEmail__c });
        msg.setSubject('Order ' + order.Name + ' confirmed');
        msg.setHtmlBody(buildBody(order));
        msg.setSaveAsActivity(true);
        msg.setWhatId(order.Id);

        Messaging.SendEmailResult[] results =
            Messaging.sendEmail(new Messaging.SingleEmailMessage[]{ msg });
        for (Messaging.SendEmailResult r : results) {
            if (!r.isSuccess()) {
                Logger.warn('Send failed', r.getErrors()[0].getMessage());
            }
        }
    }
}
```

`setSaveAsActivity(true)` plus `setWhatId(order.Id)` puts the
message on the Order's activity timeline. Without `setWhatId`,
nothing shows even with `saveAsActivity` true.

## Example 2: Render a stored template against a custom object

`setTemplateId` requires `setTargetObjectId` to be a Contact, Lead,
or User. To merge against an Account or custom object, render the
body manually:

```apex
EmailTemplate tpl = [
    SELECT Id FROM EmailTemplate WHERE DeveloperName = 'Order_Confirmation'
];
Messaging.SingleEmailMessage msg =
    Messaging.renderStoredEmailTemplate(tpl.Id, null, order.Id);

msg.setToAddresses(new String[]{ order.CustomerEmail__c });
msg.setOrgWideEmailAddressId(oweId);
msg.setSaveAsActivity(false);

Messaging.sendEmail(new Messaging.SingleEmailMessage[]{ msg });
```

The second arg (`targetObjectId`) is `null` because we do not need
Contact/Lead/User merge fields. The `whatId` (third arg) gives the
template access to the Order's fields.

## Example 3: Bulk send with per-recipient personalization

Stay under the per-call recipient cap (currently 100 per
SingleEmailMessage); split bulk sends into chunks:

```apex
public class CampaignSender {
    public static void send(List<Lead> leads, String tplDevName) {
        EmailTemplate tpl = [
            SELECT Id FROM EmailTemplate WHERE DeveloperName = :tplDevName
        ];
        OrgWideEmailAddress owe = [
            SELECT Id FROM OrgWideEmailAddress
            WHERE Address = 'campaigns@acme.com' LIMIT 1
        ];

        List<Messaging.SingleEmailMessage> batch =
            new List<Messaging.SingleEmailMessage>();
        for (Lead l : leads) {
            Messaging.SingleEmailMessage m =
                Messaging.renderStoredEmailTemplate(tpl.Id, l.Id, null);
            m.setTargetObjectId(l.Id);
            m.setOrgWideEmailAddressId(owe.Id);
            m.setSaveAsActivity(true);
            batch.add(m);
            if (batch.size() == 100) {
                Messaging.sendEmail(batch, false);
                batch.clear();
            }
        }
        if (!batch.isEmpty()) {
            Messaging.sendEmail(batch, false);
        }
    }
}
```

`allOrNone=false` lets per-recipient failures be reported in the
result list without rolling back the whole call. This matters when
one Lead has a bouncing address — the other 99 should still go.

## Example 4: Attachments without re-querying ContentVersion bytes

```apex
ContentVersion cv = [
    SELECT VersionData, Title, FileExtension, FileType
    FROM ContentVersion WHERE Id = :cvId
];

Messaging.EmailFileAttachment att = new Messaging.EmailFileAttachment();
att.setFileName(cv.Title + '.' + cv.FileExtension);
att.setBody(cv.VersionData);
att.setContentType(cv.FileType);

Messaging.SingleEmailMessage msg = new Messaging.SingleEmailMessage();
msg.setFileAttachments(new Messaging.EmailFileAttachment[]{ att });
msg.setToAddresses(new String[]{ 'recipient@acme.com' });
msg.setSubject('See attached');
msg.setPlainTextBody('Body here');

Messaging.sendEmail(new Messaging.SingleEmailMessage[]{ msg });
```

`setEntityAttachments(new String[]{ cvId })` is the alternative —
attach by ContentVersion Id without re-encoding. Both work; the
inline form is required when the bytes are computed (e.g. a PDF
generated by `PageReference.getContentAsPDF`).

## Example 5: Daily-limit-aware send

```apex
public class SafeEmailSender {
    public class DailyLimitException extends Exception {}

    public static void send(List<Messaging.SingleEmailMessage> msgs) {
        try {
            Messaging.sendEmail(msgs, false);
        } catch (System.HandledException e) {
            if (e.getMessage().contains('SINGLE_EMAIL_LIMIT_EXCEEDED')) {
                Logger.warn('Daily limit hit; queueing ' +
                            msgs.size() + ' for retry');
                EmailRetryQueue.enqueue(msgs);
                throw new DailyLimitException(
                    'Daily 5,000-email limit exceeded; retried tomorrow.'
                );
            }
            throw e;
        }
    }
}
```

The 5,000-email cap is per-org per-rolling-24h, and counts each
recipient.

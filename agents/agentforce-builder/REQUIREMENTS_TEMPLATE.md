## 4. Action shape + trust posture

- Class must declare `@InvocableMethod` with description + label so the Agent planner can reason about it.
- Method signature MUST be `public static List<Response> <name>(List<Request> requests)`.
- Inner `class Request` and inner `class Response` MUST exist; every field is decorated with `@InvocableVariable` and documented.
- Request inputs are validated and all exceptions are caught — the Response carries `success`/`errorMessage` so the Agent never sees a raw stack trace.
- Trust-layer constraints above are enforced BEFORE the action returns data: PII is masked, no unredacted tokens cross the boundary, no external callouts unless the action is explicitly annotated `callout=true`.
- Coverage floor for Gate C HIGH confidence is 85%; floor for MEDIUM is 75%.
